Critical bug that was found
- Replay snapshots were being written with current-time names like today_YYYYMMDD_HHMMSS.csv.
- Because replay loop iterations could finish within the same second, different raw historical JSONs wrote the same snapshot filename.
- That caused 10 historical raws to collapse into 7 or 8 unique snapshots.
- This also created the “weird future / wrong date” replay naming problem.

What fixed it
- Replay naming was changed in the real implementation file src/Atlas/rebuild_today_from_any_raw.py.
- Replay snapshots now use historical identity plus raw stem, e.g.:
  - replay_20260227_prizepicks_20260227_105718.csv
- Live naming was intentionally left alone.
- A curated replay batch runner now records:
  - used_snapshots.txt
  - raw_to_snapshot.tsv
- Result: one raw -> one unique replay snapshot.

What was verified after the fix
- The snapshots folder showed 10 distinct replay snapshots with historical names.
- The curated replay/backtest run finally processed all 10 intended snapshots end-to-end.
- Historical IAEL selection looked correct for each replay snapshot.
- Merge missing rates were clean (0.00%).

def _expand_directions(board: pd.DataFrame) -> pd.DataFrame:
    b = board.copy()

    if "direction" not in b.columns:
        b["direction"] = ""

    b["direction"] = _normalize_direction(b["direction"])

    already_dir = b["direction"].isin(["OVER", "UNDER"])
    directed = b.loc[already_dir].copy()
    undirected = b.loc[~already_dir].copy()

    if undirected.empty:
        return directed.reset_index(drop=True)

    over = undirected.copy()
    over["direction"] = "OVER"

    under = undirected.copy()
    under["direction"] = "UNDER"

    out = pd.concat([directed, over, under], ignore_index=True)
    return out.reset_index(drop=True)

    import pandas as pd

    df = pd.read_csv(today_path)
    meta.today_csv_path = str(today_path.resolve())
    meta.today_rows = int(len(df))

    if df.empty:
        raise BacktestV2Error("today.csv is empty")

    required = ["player_key", "stat", "line", "direction", "tier", "game_id", "game_date"]
    missing = [c for c in required if c not in df.columns]
    if missing:
        raise BacktestV2Error(f"today.csv missing required columns: {missing}")

    # Normalize the behavioral key for diagnostics only.
    diag_key = ["player_key", "stat", "line", "direction", "tier", "game_id", "game_date"]
    d = df.copy()
    for c in diag_key:
        d[c] = d[c].astype(str).fillna("").str.strip()
