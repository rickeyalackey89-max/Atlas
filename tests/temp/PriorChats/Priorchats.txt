Yes — there are a few places in the backtest file where 10 inputs can turn into 8 evaluated snapshots, and the biggest one is not a hard “stop at 8” cap.

What the file actually does:

It does not limit snapshots to 8 anywhere.

If you pass explicit --snapshot args, it first does:
snaps = sorted(set(snaps))

So if the same snapshot path is passed twice, duplicates are silently removed before scoring.

Then, for each snapshot, it builds eval_df from rows where outcome matching succeeded:
eval_df = merged.loc[merged["hit"].notna()].copy()

And this is the important silent drop:
if eval_df.empty, it does:
continue

That means a snapshot can be present in the input list, but if it produces zero matched evaluable legs against Last10.csv, it gets skipped entirely and never makes it into out_frames.

So the most likely reasons 10 became 8 are:

You did not actually pass 10 unique snapshot CSV paths
because sorted(set(snaps)) collapsed duplicates.

Two of the 10 snapshots produced zero evaluable rows
due to join/match failure on player_norm + game_date, or because those boards only produced pushes / missing actuals.

Less likely, but still possible: you only had 8 replay-generated snapshot CSVs, because the backtest script does not read the 10 raw JSONs directly. It only reads snapshot CSVs. So if you replayed 10 raws but only 8 snapshot boards were actually created/preserved, the backtest can only score 8.

The key point is this:
the script is built to silently skip empty snapshots, not fail loudly on them.
So “reading 8 instead of 10” usually means 2 snapshots entered the loop but produced no evaluated legs, not that the script has an 8-snapshot cap.

The exact lines that matter are:

dedupe of snapshot list

eval_df = ... hit.notna()

if eval_df.empty: continue

So your instinct that “something in the file is stopping it at 8” is close, but more precisely:
something in the file is silently dropping non-evaluable snapshots.

from datetime import datetime

def _build_replay_snapshot_name(raw_path: str, board_df) -> str:
    raw_stem = Path(raw_path).stem  # prizepicks_20260227_105718

    game_dates = []
    if "game_date" in board_df.columns:
        game_dates = [
            str(x).strip()
            for x in board_df["game_date"].dropna().astype(str).tolist()
            if str(x).strip()
        ]

    slate_date = None
    if game_dates:
        # assuming dates are already parseable like 2026-02-27 or 2/27/2026
        parsed = []
        for g in game_dates:
            for fmt in ("%Y-%m-%d", "%m/%d/%Y", "%-m/%-d/%Y"):
                try:
                    parsed.append(datetime.strptime(g, fmt).date())
                    break
                except Exception:
                    pass
        if parsed:
            slate_date = min(parsed).strftime("%Y%m%d")

    if not slate_date:
        # safe fallback if board content is weird
        slate_date = "unknown_date"

    return f"replay_{slate_date}_{raw_stem}.csv"
