"std_logit_p_adj_STANDARD": 0.6208364443808859" this? what is it telling us that we should do?

That value is telling you how much spread your inputs have in “log-odds space” for STANDARD legs before the kernel touches them.

What it is
std_logit_p_adj_STANDARD = std( logit(p_adj) ) over STANDARD legs.

Where logit(p) = ln(p/(1-p)).

So 0.6208 means: in log-odds space, STANDARD leg probabilities aren’t all clustered—they have a moderate amount of dispersion.

What it implies for kernel behavior
Your kernel is basically:

log(leg_factor) = a + b * logit(p_adj) (anchored so factor≈1 at p0)

So the spread of log(leg_factor) is approximately:

std(log(leg_factor)) ≈ |b| * std_logit

With your current b ≈ -0.24, that implies:

std(log(leg_factor)) ≈ 0.24 * 0.6208 ≈ 0.149

That’s a per-leg discrimination scale in log space.

Then for an n-leg slip:

std(log(kernel_mult)) ≈ sqrt(n) * 0.149

So roughly:

4-leg: ~ 2 * 0.149 = 0.298

5-leg: ~ 2.236 * 0.149 = 0.333

In practice you’re seeing smaller per-leg sigmas from built slips (because slip builder selects a subset of legs, mixes tiers, clamp effects, etc.), but that’s the theoretical ceiling given the raw leg distribution.

What should it tell us to do?
By itself: nothing to change. It’s a diagnostic, not a knob.

What it is useful for:

If std_logit is tiny (like 0.05–0.15), then the whole slate’s p_adj are bunched up, and no reasonable b will create discrimination—you’d need to fix upstream probability variance (gamelogs, role priors, clamps, etc.).

If std_logit is healthy (like your 0.62), then the slate has enough signal spread; if System still looks compressed, the compression is coming from:

kernel slope too small for System, or

slip builder selecting legs from a narrower band, or

clamp rails / tier mix collapsing variance downstream.

So for you, 0.6208 is basically saying:

“Inputs have plenty of spread; if outputs are still compressed, it’s not because p_adj is flat.”

That’s consistent with why your slip-space sigma ratio method is the right guide.

Why the “from_logit” b suggestion can mislead
Oracle’s old solve does: b ≈ -target_sigma_leg / std_logit.

But your target sigma_leg is measured from built slips, not from the raw pool. Built slips concentrate the distribution, so using raw std_logit can under-suggest |b|. That’s exactly what you observed.

if iael_doc is None:
    iael_audit["error"] = f"missing_or_unreadable:{iael_path}"
    out["iael_audit"] = iael_audit
else:
    # meta (best-effort; tolerate schema drift)
    meta = {}
    for k in ["report_label", "pulled_at_local", "url", "source_url", "report_date", "report_time"]:
        if k in iael_doc:
            meta[k] = iael_doc.get(k)

    teams_with_outs = sorted({r.get("team") for r in hard_rows if r.get("team")})
    iael_audit["teams_with_hard_invalids"] = teams_with_outs

        outs_used_col = _find_col(legs, ["role_ctx_outs_used", "role_ctx_outs", "outs_used"])

        prole_col = _find_col(legs, ["p_role"])
        player_col = _find_col(legs, ["player", "name"])
        stat_col = _find_col(legs, ["stat", "stat_type"])
        tier_col = _find_col(legs, ["tier"])
        dir_col = _find_col(legs, ["direction", "side"])

        iael_audit["cols"] = {

            "outs_used": outs_used_col,
            "p_adj": padj_col,
            "p_role": prole_col,
            "player": player_col,
            "stat": stat_col,
            "tier": tier_col,

            team_cov_sorted = sorted(

                    if c and c in miss_df.columns and c not in show_cols:

                # top 50 overall by p_adj
                miss_top = miss_df.sort_values("_p_adj_num", ascending=False).head(50)

            # stdout-friendly short summary
            try:

                print(f"[IAEL] Loaded teams_with_outs={len(teams_with_outs)} cutoff_p_adj={IAEL_MISS_P_ADJ_CUTOFF}")

def _iael_team_to_abbrev(team: str | None) -> str | None:
    if not team:
        return None
    t = str(team).strip()

    # IAEL seems to use CamelCase full names like "BostonCeltics"
    # Map those to standard NBA abbreviations used in scored_legs_deduped ("BOS", "DAL", etc.)
    MAP = {
        "AtlantaHawks": "ATL",
        "BostonCeltics": "BOS",
        "BrooklynNets": "BKN",
        "CharlotteHornets": "CHA",
        "ChicagoBulls": "CHI",
        "ClevelandCavaliers": "CLE",
        "DallasMavericks": "DAL",
        "DenverNuggets": "DEN",
        "DetroitPistons": "DET",
        "GoldenStateWarriors": "GSW",
        "HoustonRockets": "HOU",
        "IndianaPacers": "IND",
        "LAClippers": "LAC",
        "LosAngelesClippers": "LAC",
        "LALakers": "LAL",
        "LosAngelesLakers": "LAL",
        "MemphisGrizzlies": "MEM",
        "MiamiHeat": "MIA",
        "MilwaukeeBucks": "MIL",
        "MinnesotaTimberwolves": "MIN",
        "NewOrleansPelicans": "NOP",
        "NewYorkKnicks": "NYK",
        "OklahomaCityThunder": "OKC",
        "OrlandoMagic": "ORL",
        "Philadelphia76ers": "PHI",
        "PhoenixSuns": "PHX",
        "PortlandTrailBlazers": "POR",
        "SacramentoKings": "SAC",
        "SanAntonioSpurs": "SAS",
        "TorontoRaptors": "TOR",
        "UtahJazz": "UTA",
        "WashingtonWizards": "WAS",
    }

    # quick direct hit
    if t in MAP:
        return MAP[t]

    # tolerate spacing/underscores/hyphens
    t2 = t.replace(" ", "").replace("_", "").replace("-", "")
    if t2 in MAP:
        return MAP[t2]

    # if IAEL ever returns abbrev already, pass through
    if len(t2) == 3 and t2.isalpha():
        return t2.upper()

    return None

def _iael_is_impact_out(r: dict) -> bool:
    # Conservative: only treat as "impact" when it looks like a real injury/availability item,
    # not a G-League assignment / two-way churn.
    reason = str(r.get("reason") or "").lower()
    status = str(r.get("status") or "").lower()
    tag_prob = bool(r.get("tag_probable") is True)

    # Ignore "probable" tags
    if tag_prob or status == "probable":
        return False

    # Exclude obvious non-impact categories
    NON_IMPACT = [
        "gleague", "g league", "two-way", "two way", "assignment", "on assignment",
        "g-league two-way", "g league two way"
    ]
    if any(k in reason for k in NON_IMPACT):
        return False

    # Include common impact categories
    IMPACT = [
        "injury", "illness", "soreness", "sprain", "strain", "fracture", "concussion",
        "rest", "suspension", "not with team", "personal", "family", "return to competition",
        "conditioning", "back", "ankle", "knee", "hamstring", "groin", "foot", "wrist", "shoulder"
    ]
    if any(k in reason for k in IMPACT):
        return True

    # Fallback: if it's hard_invalid and not explicitly non-impact, treat as impact
    return True
